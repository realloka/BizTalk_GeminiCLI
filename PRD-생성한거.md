# PRD: BizTone Converter - 업무 말투 자동 변환 솔루션

> 이 문서는 'BizTone Converter' 프로젝트의 제품 요구사항을 정의합니다. 프로젝트의 목표, 기능, 제약 조건 등을 명확히 하여 모든 이해관계자가 일관된 방향으로 프로젝트를 이해하고 진행하는 것을 목표로 합니다.

**버전**: 1.0
**작성일**: 2026-02-20
**최종 수정일**: 2026-02-20

---

## 1. 개요 (Overview)

### 1.1. 문제 정의 (Problem)
현대 업무 환경에서 신입사원이나 비즈니스 커뮤니케이션에 미숙한 직원들은 상사, 동료, 고객 등 다양한 대상에게 적절한 어조와 말투를 사용하는 데 어려움을 겪습니다. 이는 커뮤니케이션의 오해를 유발하고, 메시지 작성에 불필요한 시간을 소모시켜 업무 효율성을 저하시키는 원인이 됩니다.

### 1.2. 해결 방안 (Solution)
**BizTone Converter**는 사용자의 일상적인 표현을 전문적인 비즈니스 말투로 자동 변환해주는 웹 기반 솔루션입니다. 대화 상대(상사, 동료, 고객)에 따라 최적화된 어투를 제안하여, 명확하고 효과적인 비즈니스 커뮤니케이션을 지원하고 사용자의 업무 생산성을 향상시킵니다.

### 1.3. 비전 (Vision)
누구나 쉽고 자신감 있게 비즈니스 커뮤니케이션을 수행할 수 있는 환경을 제공하여, 조직 내외부의 소통 품질을 한 단계 끌어올리는 것을 목표로 합니다.

---

## 2. 목표 사용자 (Target Audience)

- **주요 사용자 (Primary)**
  - **신입사원 및 주니어 직원**: 비즈니스 커뮤니케이션 스킬 향상이 필요한 사회초년생.
  - **외부 고객 담당자**: 고객과의 공식적이고 정중한 소통이 필수적인 영업, CS, 마케팅 직군.

- **부차적 사용자 (Secondary)**
  - **팀 관리자**: 여러 팀원 및 타 부서와의 원활한 소통을 원하는 리더.
  - **외국인 직원**: 한국의 비즈니스 문화와 언어에 익숙하지 않은 직원.

## 3. 기능 요구사항 (Functional Requirements)

### 3.1. 핵심 기능
| ID | 기능명 | 설명 | 우선순위 |
|---|---|---|---|
| F-01 | **대상별 말투 변환** | 사용자가 '상사', '타팀 동료', '고객' 중 변환 대상을 선택할 수 있다. | Must-have |
| F-02 | **실시간 텍스트 변환** | 사용자가 원본 텍스트를 입력하고 '변환' 버튼을 누르면, AI 모델을 통해 변환된 텍스트를 즉시 제공한다. | Must-have |
| F-03 | **결과 비교 뷰** | 원본 텍스트와 변환된 텍스트를 좌우로 나란히 비교하여 보여준다. | Must-have |
| F-04 | **결과 복사** | 변환된 텍스트를 클립보드에 복사하는 버튼을 제공한다. | Must-have |
| F-05 | **피드백 수집** | 변환 결과에 대해 '도움됨'/'도움안됨' 형태의 간단한 피드백을 수집할 수 있다. | Should-have |

### 3.2. 사용자 인터페이스 (UI/UX)
| ID | 기능명 | 설명 | 우선순위 |
|---|---|---|---|
| UI-01 | **반응형 웹 디자인** | 데스크톱, 태블릿, 모바일 등 다양한 디바이스에서 최적화된 UI를 제공한다. | Must-have |
| UI-02 | **직관적인 입력 폼** | 단일 페이지 내에서 텍스트 입력, 대상 선택, 결과 확인이 모두 가능해야 한다. | Must-have |
| UI-03 | **글자 수 카운터** | 텍스트 입력 영역에 현재 입력된 글자 수를 실시간으로 표시한다. (최대 500자) | Should-have |
| UI-04 | **입력 가이드 (도구팁)** | 각 대상별로 어떤 내용을 입력하면 좋은지 예시를 보여주는 도구팁(Tooltip)을 제공한다. | Could-have |
| UI-05 | **결과 하이라이트** | 원본과 비교하여 말투, 어휘 등 주요 변경 사항을 시각적으로 강조하여 보여준다. | Could-have |

### 3.3. AI 변환 엔진 (Backend)
| ID | 기능명 | 설명 | 우선순위 |
|---|---|---|---|

## 4. 비기능 요구사항 (Non-Functional Requirements)

| ID | 항목 | 요구사항 |
|---|---|---|
| NF-01 | **성능 (Performance)** | - 텍스트 변환 요청 시, 평균 응답 시간이 3초 이내여야 한다. (p95 기준) |
| NF-02 | **가용성 (Availability)** | - 서비스는 99% 이상의 가용성을 목표로 한다. |
| NF-03 | **보안 (Security)** | - Groq AI API 키 등 민감 정보는 환경 변수로 관리하며, 클라이언트에 노출되어서는 안 된다. <br> - 모든 통신은 HTTPS를 통해 암호화한다. |
| NF-04 | **접근성 (Accessibility)** | - WCAG 2.1 AA 수준을 목표로 키보드 네비게이션, 스크린 리더기 지원 등 웹 접근성을 준수한다. |
| NF-05 | **확장성 (Scalability)** | - 향후 새로운 변환 대상(e.g., '투자자') 추가나 기능 확장이 용이한 구조로 설계한다. <br> - Vercel 서버리스 아키텍처를 통해 트래픽 증가에 따른 자동 확장을 지원한다. |

## 5. 성공 지표 (Success Metrics)

- **사용자 만족도**: 변환 결과에 대한 '도움됨' 피드백 비율 (80% 이상 목표)
- **서비스 활성화**: 일일 활성 사용자 수 (DAU) 및 월간 활성 사용자 수 (MAU)
- **핵심 기능 사용률**: 일일 텍스트 변환 요청 수
- **성능**: 평균 응답 시간 3초 미만 유지
